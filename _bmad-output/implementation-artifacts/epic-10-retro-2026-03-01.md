## ✅ What Went Well

- **"Shared Offset" Architectural Breakthrough:** While implementing Story 10.3, the team identified and resolved brittle layout couplings by centralizing horizontal and vertical rhythm tokens in `globals.css`. By calculating **Collision Offsets** (76px `left-filter-offset`) as design variables, we achieved pixel-perfect alignment that is now robust against container changes.
- **The "About" Pivot & Content Synthesis:** Successfully transitioned the "About this Project" section from a standalone page to a high-density, row-pivoted modal (Story 10.1). While it required significant content/UX iteration to find the right structure, the resulting V1 is polished, stable, and ready for external evaluators.
- **High-Fidelity Theme Sync:** Achieved 100% visual consistency between Light and Dark modes (Story 10.2). By moving away from hardcoded zinc-900 palettes to semantic tokens (`bg-background`, `prose-zinc dark:prose-invert`), the "Tinted Neutrality" aesthetic now feels premium regardless of theme preference.
- **Muted Reset UX:** The revamped "Clear Filter" workflow (Story 10.3) successfully introduced a less intrusive, muted style that prioritizes layout stability over visual "noise," despite a long trial-and-error cycle on positioning.

## 💡 Key Learnings

### CSS Variables as Architecture
- **Insight:** Hardcoding pixel offsets (like `-mt-28` or `pl-20`) in components leads to "magic number drift" that is impossible to maintain at scale.
-   **Insight:** Hardcoding pixel offsets (like `-mt-28` or `pl-20`) in components leads to "magic number drift" that is impossible to maintain at scale.
-   **Takeaway:** Layout rhythm is a **Global Concern**. By using `--spacing-header-mb` and `--spacing-filter-collision`, we moved from "guessing the gap" to "calculating the overlap." This pattern should be standard for any future grid-based components.

### Modal Density Over Layout Breadth
-   **Insight:** Long, scrolling vertical columns in a modal feel "heavy" and disconnected.
-   **Takeaway:** The **Stacked Row Architecture** implemented in the About modal (Philosophy -> Operations -> Activity) creates a much more scannable and "premium" narrative flow. Quantitative metrics should always be paired with high-impact grid rows to maximize scannability.

### The "Magic Number" Trap vs. Computed Intent
-   **Insight:** We spent too long trial-and-erroring the "Clear Filter" position because we weren't thinking about the *intentional relationship* between the header and the filter bar. We were trying to "pixel-push" it into place instead of "architecting" it into place.
-   **Takeaway:** Brittleness occurs when we use absolute values to solve relative problems. The fix (Story 10.3 follow-up) was moving the "Intent" (collision offsets) into the CSS variables. In the future, if a developer finds themselves using negative margins or hardcoded `pl-76` to "line things up," it's a signal to stop and call for an architectural review.
