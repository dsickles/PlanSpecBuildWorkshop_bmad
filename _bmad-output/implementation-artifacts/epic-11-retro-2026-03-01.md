## 💡 Key Learnings

### Metadata Driver Precedence
- **Insight:** Using `index.md` as the primary driver for both project-level and shared-branding metadata ensures that the file system structure remains secondary to the author's intent.
- **Takeaway:** This pattern simplifies the addition of new "sectors" (like `_shared`) without requiring logic changes in the content parser.

### Intentional UI Silence
- **Insight:** Over-communicating state in fallbacks (e.g., showing "Concept" pills for empty groups) creates visual clutter.
- **Takeaway:** "Quieting" empty states through fallback cards without status pills maintains the structural rhythm and professional polish of the dashboard.

### Recursive Documentation Alignment
- **Insight:** Implementing the `sourcePath` feature in Epic 9 was only half the battle; formalizing it in Epic 11's architecture sync made it a stable, maintainable project standard.
- **Takeaway:** Features aren't truly complete until they are documented as canonical patterns.
