## 💡 Key Learnings (from Code Reviews)

### UX Pivot: State Persistence
- **Insight:** Originally, we planned to collapse all document rows when clearing filters. Review feedback showed that this felt like "losing work."
- **Decision:** Document rows now **maintain their expanded state** across filter changes, preserving user context. This was formalized in `architecture.md`.

### Technical Hardening: Build Environment Resilience
- **Insight:** Path resolution logic in `content-parser.ts` was initially brittle and prone to failure in serverless environments.
- **Fix:** Refactored the ingestion engine to pre-resolve canonical project titles and handle relative paths more robustly, de-risking the upcoming Vercel deployment.

### Quality: Test Assertion Precision
- **Insight:** Early unit tests were too shallow (checking for presence only).
- **Fix:** Pushed for high-precision assertions in `metrics.test.ts` to verify the exact mathematical rollups of the portfolio content.
