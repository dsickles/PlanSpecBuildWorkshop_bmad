## ✅ What Went Well

- **Centralized Content Control:** Migrated all display ordering from fragmented frontmatter fields to a single `sort-config.yaml` manifest. This provides an unambiguous, author-first mechanism for UI presentation.
- **DevEx & Validation:** Implemented a JSON schema for `sort-config.yaml` along with a recommended VS Code extension. This significantly improved the developer experience by providing real-time validation and autocompletion for content authors.
- **Outlier Management:** Successfully integrated an `additional` projects section in the manifest, allowing for flexible project categorization without forcing every item into a primary hierarchy.
- **Cleanup & Decoupling:** Purged the legacy `order` frontmatter field and the hardcoded `PINNED_PROJECT_SLUG` logic, reducing technical debt and "magic string" dependencies in the ingestion engine.

---

## 💡 Key Learnings

### Type Refactoring & Registry regressions
- **Insight:** Moving the core `ParsedArticle` and related types to a central `schema.ts` was architectural debt that needed solving, but it triggered wide-spread import regressions across UI components and unit tests.
- **Takeaway:** Core type refactors in a mature-ish codebase require a proactive "sweep" of all consumers (grep is your friend) rather than relying solely on TypeScript's immediate feedback during a single-file edit.

### Automated Testing Guardrails
- **Insight:** Integrated "Semantic Test Tokens" (`data-testid`, `aria-label`) into `FallbackCard` and `BlueprintErrorRow` to resolve dashboard test collisions identified in Epic 5.
- **Outcome:** Normalizing fallback labels (e.g., `aria-label="agent content unavailable"`) eliminated flaky test failures caused by generic text assertions across different columns.
