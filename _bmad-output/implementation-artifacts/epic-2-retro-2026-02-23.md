## ⚠️ What Needs Improvement

### Pattern 1: Gap Between UX Intent and Architecture Data Mapping
The UX Mockup explicitly showed the "Blueprints" column organized into grouped Project rows with document counts. However, `architecture.md` didn't explicitly mandate how this relational mapping should occur at the component layer. As a result, the devs had to build the `docsByProject` grouping logic (`Map<string, ParsedArticle[]>`) on the fly in the Server Component during Story 2.2.

**Fix for future epics:** Before development begins on complex UI state (like Epic 3 filtering), the Architecture document must define the explicit State Mapping.

### Pattern 2: Component Prop vs Zod Schema Mismatch
The Zod Frontmatter schema enforced `external_url` (snake_case), but the React component expected `externalUrl` (camelCase). This caused mild friction in Story 2.1's code review.

**Fix for future epics:** When defining prop interfaces, ensure 1:1 parity with the validated Zod schema shape or explicitly document the transformation layer.
