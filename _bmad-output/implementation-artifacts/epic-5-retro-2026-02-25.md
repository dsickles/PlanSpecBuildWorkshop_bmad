## 💡 Key Learnings (from 'Gnarly' Code Review)

### Testing: The String Collision Bottleneck
- **Insight:** During Story 5.3, the team faced a significant delay due to a test assertion mismatch. Generic text labels like "Content unavailable" were used in both `FallbackCard` (Agents) and `BlueprintErrorRow` (Docs), causing `screen.getAllByText` to return unexpected counts.
- **Root Cause:** Over-reliance on generic UI text for test assertions and a stubborn Jest cache that masked real data discrepancies.
- **Resolution:** The Project Lead's intervention (suggesting `screen.debug()`) allowed the team to see the actual rendered HTML and realize the mental model of the mock data was misaligned with the global selector.
