## 💡 Key Learnings

### Design-Driven Discovery Trade-offs
- **Insight:** Implementing the modal without a pre-frozen UX spec allowed for a better final product (e.g., discovering the need for adaptive ToC centering) but increased implementation friction and velocity predictability.
- **Takeaway:** Discovery-driven development is excellent for 'crown jewel' features where the 'feel' is the product, but should be managed with shorter check-in loops to catch technical regressions (like scroll jitter) earlier.

### Technical & Code Review Themes
- **Unique Identifier Integrity:** Duplicate headers in markdown require sequential slug suffixing (e.g., `overview-1`) to prevent ToC anchoring collisions.
- **Layered Logic Boundaries:** Keeping the `toc-engine.ts` separate from the React view prevented the UI layer from becoming brittle as the parsing logic grew complex.
- **Accessibility as a First-Class Citizen:** Code reviews identified gaps in ARIA roles for the reading progress bar, reinforcing the need for automated accessibility checks in the component suite.
